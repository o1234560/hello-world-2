# 打包发布

## 生成一个签名密钥

你可以用`keytool`命令生成一个私有密钥。

`keytool -genkeypair -v -storetype PKC312 -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -kyesize 2048 -validity 10000`

```shell
keytool -genkeypair -v -storetype PKC312 -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -kyesize 2048 -validity 10000
```

这条命令会要求你输入密钥库（keystore）和对应密钥的密码，然后设置一些发行相关的信息。最后它会生成一个叫做`my-release-key.keystore`的密钥库文件。

## 设置 gradle 变量

1. 把`my-release-key.keystore`文件放到项目文件夹的`android/app`文件夹中。

2. 全局配置`C:/User/用户名/.gradle/gradle.properties`；项目配置`项目目录/android/gradle.properties`。

```properties
MYAPP_RELEASE_STORE_FILE=my-release-key.keystore
MYAPP_RELEASE_KEY_ALIAS=my-key-alias
MYAPP_RELEASE_STORE_PASSWORD=密码
MYAPP_RELEASE_KEY_PASSWORD=密码
```

上面的这些会作为 gradle 的变量，在后面的步骤中可以用来给应用签名。

## 把签名配置加入到项目的 gradle 配置中

项目文件夹中的`android/app/build.gradle`，添加如下的签名配置：

```gradle
...
android {
    ...
    defaultConfig { ... }
    signingConfigs {
        release {
            if (project.hasProperty('MYAPP_RELEASE_STORE_FILE')) {
                storeFile file(MYAPP_RELEASE_STORE_FILE)
                storePassword MYAPP_RELEASE_STORE_PASSWORD
                keyAlias MYAPP_RELEASE_KEY_ALIAS
                keyPassword MYAPP_RELEASE_KEY_PASSWORD
            }
        }
    }
    buildTypes {
        release {
            ...
            signingConfig signingConfigs.release
        }
    }
}
...
```

## 生成发行 APK 包

在终端运行以下命令：

```shell
cd android
./gradlew assmbleRelease
```

执行脚本文件`gradlew`，运行参数为`assmbleRelease`，把所有用到的 Javascript 代码打包到一起，然后内置到 APK 包中。

生成的 APK 文件位于`android/app/build/outputs/app/release/app-release.apk`。
